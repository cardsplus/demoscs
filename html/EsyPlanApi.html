Überblick

Der Dienst hat die Aufgabe, Projekte zu verwalten. Projekte haben einen verantwortlichen Nutzer, den Besitzer. Beliebig viele Nutzer (inkl. des Besitzer) sind Projektmitglieder, die Aufgaben in diesem Projekt erledigen.

Der Dienst ist Teil eines Self Contained Systems.

Siehe https://scs-architecture.org für mehr Informationen.

Der Dienst nutzt das Spring-Framework.

Siehe https://spring.io/projects für mehr Informationen.

Der Dienst nutzt das Svelte-Framework.

Siehe https://svelte.dev/ für mehr Informationen.

Der Dienst zeigt an einem nicht trivialen Beispiel den effektiven Einsatz des Spring-Frameworks und weiterer Projekte aus den Spring-Universum.

Modell

Das Datenmodell wird nach den Prinzipien des Domain Driven Design entwickelt.

Siehe https://cardsplus.info/de/cards-und-ddd für mehr Informationen.

Das Modell ist Teil der gemeinsamen Sprache. Aufgrund der Entscheidung, Deutsch als Domänensprache festzulegen, werden konsequent wichtige Begriffe der Domäne aus einem Glossar auch im Code verwendet. Technische Ausdrücke bleiben jedoch wie gewohnt in englischer Sprache.

Das Modell ist so gestaltet, dass die Daten in einer relationalen Datenbank dauerhaft gespeichert werden. Für die Speicherung in der Datenbank wird spring-data-jpa als Abstraktion eingesetzt.

Siehe https://spring.io/projects/spring-data-jpa für mehr Informationen.

Optional kann jeder gespeicherte Datensatz auch als Datensatz in einem REST-API veröffentlicht werden. Für die Veröffentlichung der Daten in einem REST-API wird spring-data-rest als Abstraktion eingesetzt.

Siehe https://spring.io/projects/spring-data-rest für mehr Informationen.

Optional kann jeder gespeicherte Datensatz auch als Nachricht in einem Datenstrom veröffentlicht werden. Für die Veröffentlichung der Daten in einem Datenstrom wird spring-kafka als Abstraktion eingesetzt.

Siehe https://spring.io/projects/spring-kafka für mehr Informationen.

Anwendung

Der Dienst hat eine reaktive browser-basierte Benutzeroberfläche. Der Datenzugriff wird über eine REST-Schnittstelle mit HATEOAS realisiert.

Siehe https://spring.io/projects/spring-hateoas für mehr Informationen.

Projekt-API

REST-API für die Verwaltung von Projekt-Objekten.

POST /api/projekt

Die Operation speichert ein neues Projekt-Objekt.

CURL
$ curl 'http://localhost:8080/api/projekt' -i -X POST \
    -H 'Content-Type: application/json' \
    -H 'Accept: application/json' \
    -d '{
  "name" : "Projekt A",
  "aktiv" : "true",
  "sprache" : "EN"
}'
